Introduction¶

The devicely package is made for reading, writing and anonymizing (timeshifting) health sensor data from

  • Empatica E4 (Firmware 2.1.0.4911)

  • Biovotion Everion (Firmware 03.06)

  • 1-lead ECG monitor FarosTM 180 from Bittium (Firmware 3.5.1)

  • Spacelabs (SL 90217)

  • Tags (obtained from the app: TimeStamp for Android Version 1.36)

  • Shimmer Consensys GSR (Shimmer3 GSR Development Kit)

Writing the data back to csv files. The written data is organized in the same way as the read data. Thus it can be read by another EmpaticaReader.

>>> empatica_reader.write('write_path')

class devicely.empatica.EmpaticaReader(path)¶

Parses, anomnymizes and writes data generated by Empatica E4

start_times¶

contains the timestamp of the first measurement for all measured signals (BVP, ACC etc.)

Type

dict

sample_freqs¶

contains the sampling frequencies of all measured signals in Hz.

Type

dict

IBI¶

Contains inter-beat interval data with columns “timedelta” and “ibi”, indexed by time of measurement. The column “timedelta” contains the passed time in seconds since the first measurement and “ibi” contains the inter-beat interval in seconds since the last measurement.

Type

DataFrame

ACC¶

Contains the data measured with the onboard MEMS type 3-axis accelerometer, indexed by time of measurement.

Type

DataFrame

BVP¶

Contains Blood Volume Pulse data, indexed by time of measurement.

Type

DataFrame

EDA¶

Contains data captured from the Electrodermal Activity sensor, indexed by time of measurement.

Type

DataFrame

HR¶

Contains Heart Rate data, indexed by time of measurement.

Type

DataFrame

TEMP¶

Contains temperature data, indexed by time of measurement.

Type

DataFrame

data¶

Joined dataframe of all individual signal dataframes (see above). May contain NaN values because sampling frequencies differ across signals.

Type

DataFrame

timeshift(shift='random')¶

Anonymizes the data by shifting all time related columns as well as the starting_times dict.

Parameters

shift (None/'random', pd.Timestamp or pd.Timedelta) –

If shift is not specified, shifts the data by a random time interval between one month and two years to the past.

If shift is a timdelta, shifts the data by that timedelta.

If shift is a timestamp, shifts the data such that the earliest entry is at that timestamp and the remaining values keep the same time distance to the first entry.

write(path)¶

Write the signal dataframes back to individual csv files formatted the same way as the read csv files.

Parameters

path (str) –

Path of the directory in which the csv files are created.

If the directory exists, the csv files are written using writing mode ‘w’ ignoring other files in the directory.

If the directory doe not exist, it will be created.

class devicely.spacelabs.SpacelabsReader(path)¶

Parses, anomnymizes and writes data generated by Spacelabs (SL 90217)

data¶

DataFrame with read values

Type

DataFrame

subject¶

Contains the contestant’s id. Can be changed for deanonymization.

Type

str

metadata¶

The measurements’ metadata. read from the xml at the bottom of the read abp file. Can be changed for deanonymization.

Type

dict

drop_EB()¶

Drops all entries with “EB”-errors from the DataFrame

Note

Before dropping, the dataframe has a range index because the timestamps might not be unique. After dropping, the timestamp column will be unique and is thus used as an index for easy indexing.

timeshift(shift='random')¶

Anonymizes the data by shifting all time related columns.

Parameters

shift (None/'random', pd.Timestamp or pd.Timedelta) –

If shift is not specified, shifts the data by a random time interval between one month and two years to the past.

If shift is a timdelta, shifts the data by that timedelta.

If shift is a timestamp, shifts the data such that the earliest entry is at that timestamp and the remaining values keep the same time distance to the first entry.

write(path)¶

Writes the (deanonymized) DataFrame, subject id and metadata to the writing path in the same format as it was read.

Parameters

path (str) – path to writing csv. Writing mode: ‘w’
