Introduction¶

The devicely package is made for reading, writing and timeshifting health sensor data from:

  • Empatica E4 (Firmware 2.1.0.4911)

  • Biovotion Everion (Firmware 03.06)

  • 1-lead ECG monitor FarosTM 180 from Bittium (Firmware 3.5.1)

  • Spacelabs (SL 90217)

  • Tags (obtained from the app: TimeStamp for Android Version 1.36)

  • Shimmer Consensys GSR (Shimmer3 GSR Development Kit)

Examples¶

Empatica¶

The Empatice E4 wristband can be used to obtain data from inter-beat intervals, electrodermal activity, heart rate, temperature and blood volume pulse. The wristband outputs its measurements in this directory structure.

Create an EmpaticaReader object, passing as a parameter a path of the same directory structure:

>>> import devicely
>>> empatica_reader = devicely.EmpaticaReader('path/to/empatica/dir')

Accessing the individual signal dataframes:

>>> empatica_reader.ACC.head(3)
                           acc_x  acc_y  acc_z    acc_mag
2020-10-16 17:04:29.000000     -3     11     60  61.073726
2020-10-16 17:04:29.031250     -3     11     60  61.073726
2020-10-16 17:04:29.062500     -2     11     60  61.032778

You can also access empatica_reader.IBI (inter-beat intervals), empatica_reader.EDA (electrodermal activity), empatica_reader.HR (heart rate), empatica_reader.TEMP (temperature) and empatica_reader.BVP (blood volume pulse).

An EmpaticaReader object also contains a dataframe of all signals, joined by time of sampling. Because of varying sampling frequencies across signals, expect to see a lot of nan values.

>>> empatica_reader.data.head(3)
                           bvp  eda  hr   temp  acc_x  acc_y  acc_z    acc_mag  timedelta  ibi
2020-10-16 17:04:29.000000 -0.0  0.0 NaN  31.13   -3.0   11.0   60.0  61.073726        NaN  NaN
2020-10-16 17:04:29.015625 -0.0  NaN NaN    NaN    NaN    NaN    NaN        NaN        NaN  NaN
2020-10-16 17:04:29.031250 -0.0  NaN NaN    NaN   -3.0   11.0   60.0  61.073726        NaN  NaN

You can access the sampling frequency of each signal:

>>> empatica_reader.sample_freqs
{'bvp': 64.0, 'eda': 4.0, 'hr': 1.0, 'temp': 4.0, 'acc': 32.0}

Change the data by timeshifting the time of sampling by a random interval between one month and two years to the past. If you would like to keep control over the shifting interval, you can provide either a pandas.Timedelta or a pandas.Timestamp object as a parameter to EmpaticaReader.timeshift().

>>> empatica_reader.start_times
{'bvp': Timestamp('2020-10-16 17:04:29'),
 'eda': Timestamp('2020-10-16 17:04:29'),
 'hr': Timestamp('2020-10-16 17:04:39'),
 'temp': Timestamp('2020-10-16 17:04:29'),
 'acc': Timestamp('2020-10-16 17:04:29'),
 'ibi': Timestamp('2020-10-16 17:04:29')}

>>> empatica_reader.timeshift()

>>> empatica_reader.start_times
{'bvp': Timestamp('2020-03-07 03:02:16.693659368'),
'eda': Timestamp('2020-03-07 03:02:16.693659368'),
'hr': Timestamp('2020-03-07 03:02:26.693659368'),
'temp': Timestamp('2020-03-07 03:02:16.693659368'),
'acc': Timestamp('2020-03-07 03:02:16.693659368'),
'ibi': Timestamp('2020-03-07 03:02:16.693659368')}

Write the data back to csv files. The written data keeps the format of the original data, so you can use an EmpaticaReader to read your written data.

>>> empatica_reader.write('write_path')
>>> os.listdir('write_path')
['HR.csv', 'BVP.csv', 'TEMP.csv', 'EDA.csv', 'tags.csv', 'ACC.csv', 'IBI.csv']

SpaceLabs¶

Look here for an example file created by the Spacelabs ABPM monitoring system.

Creating a SpacelabsReader object, passing such a file as a parameter:

>>> spacelabs_reader = devicely.SpacelabsReader('path/to/spacelabsfile.abp')

Accessing the data:

               'NURSETECH': 'admin',

'005F3'
>>> spacelabs_reader.data.head(6)
            timestamp        date      time  SYS(mmHg)  DIA(mmHg)     x     y   z error
0 2019-03-01 16:18:00  2019-03-01  16:18:00        107         76  78.0  78.0 NaN   NaN

5 2019-03-01 16:31:00  2019-03-01  16:31:00         11          0   0.0   0.0 NaN    EB

EB-errors can be dropped using drop_EB.

The method set_window will create a timedelta window around the blood pressure measurement (e.g. 30 secs) The type of window is defined by: bfill (before the start of the measurement), bffil (half before and half after) of ffill (after the start of the measurement)

Change the data by timeshifting the time of sampling by a random interval between one month and two years to the past. If you would like to keep control over the shifting interval, you can provide either a pandas.Timedelta or a pandas.Timestamp object as a parameter to SpaceLabsReader.timeshift().

2019-03-01 16:22:00  2019-03-01  16:22:00        100         68  64.0  64.0 NaN   NaN

Write the data back to disk. The written data keeps the same format as the original file, so you can use a SpacelabsReader to read it again.

>>> spacelabs_reader.write('spacelabs_write_file.abp')
>>> 'spacelabs_write_file.abp' in os.listdir()

Tags¶

Tags are timestamps that are important to the measuring process, like the start and finishing time of measurements. The tag files can be obtained using the Android App TimeStamp.

Read and access tags:

>>> tag_reader = devicely.TagReader('path/to/readfile')
>>> tag_reader.data.head(3)
                     tag_number                tag
time
2019-03-01 16:16:37           1              Shake
2019-03-01 16:17:43           2              Start
2019-03-01 16:18:20           3     BP Measurement

Change the data by timeshifting the time of sampling by a random interval between one month and two years to the past. If you would like to keep control over the shifting interval, you can provide either a pandas.Timedelta or a pandas.Timestamp object as a parameter to TagReader.timeshift().

>>> tag_reader.timeshift()
>>> tag_reader.data.head(3)
                     tag_number                tag
time
2018-07-25 01:48:47           1              Shake
2018-07-25 01:49:53           2              Start
2018-07-25 01:50:30           3     BP Measurement

Write the data to file:

>>> tag_reader.write('path/to/writingfile')
>>> 'writingfile' in os.listdir('path/to')
True

timeshift(shift='random')¶

Shift all time related columns as well as the starting_times dict.

Parameters

shift (None/'random', pd.Timestamp or pd.Timedelta) –

If shift is not specified, shifts the data by a random time interval between one month and two years to the past.

If shift is a timdelta, shifts the data by that timedelta.

If shift is a timestamp, shifts the data such that the earliest entry is at that timestamp and the remaining values keep the same time distance to the first entry.

write(path)¶

Write the signal dataframes back to individual csv files formatted the same way as the read csv files.

Parameters

path (str) –

Path of the directory in which the csv files are created.

If the directory exists, the csv files are written using writing mode ‘w’ ignoring other files in the directory.

If the directory doe not exist, it will be created.

Raises

Exception – In case path is not specified.

drop_EB()¶

Drops all entries with “EB”-errors from the DataFrame

Note

Before dropping, the dataframe has a range index because the timestamps might not be unique. After dropping, the timestamp column will be unique and is thus used as an index for easy indexing.

set_window(window_duration, window_type)¶

Set a window around, before or after the blood pressure measurement by creating two new columns with the window_start and window_end times.

Parameters

  • window_size (pd.Timedelta, datetime.timedelta) – Duration of the window.

  • window_type (bffill, bfill, ffill) –

    Bffill stands for backward-forward fill. The window is defined as half after and half before the start of the measurement.

    Bfill stands for backward fill. The window is defined before the start of the measurement.

    Ffill stands for forward fill. The window is defined after the start of the measurement.
